# WAN DELI 改善指示書（比較ページ用）

## 現在の状態
- プレビュー本体は実コード版。モック画像の重ね・文字入り背景は禁止。
- 背景は文字なしの固有ファイル名素材を使用する。
- 現在のPC比較スコア: 約87.76%。95%未達。

## 最大差分と改善方針

### 1. 右側メインビジュアル（最優先）
**差分**
- モック: 犬の顔・胴体・商品群・植物・白いボウルが、右側に密度高くまとまっている。
- 実装: 犬と商品はあるが、商品配置・ボウル・犬の大きさ・植物の位置がモックと一致していない。
- ここが差分マスクで最も大きい黒白差分になっている。

**再現する内容**
- 文字なし背景素材をImage2.0で再生成する。
- 指示は「モックの右側構図を維持」「左側は空白」「文字・ロゴ・ボタン・UIは絶対に入れない」。
- ファイル名は必ず固有名にする。`hero-visual.png` の使い回しは禁止。

**実装指示**
- CSSの `.hero-photo` の `background-position` / `background-size` を調整し、犬の顔がモックと同じx/yに来るまで比較する。
- 背景画像変更後は必ずPlaywrightで実URLを撮影する。

### 2. 左テキストブロック
**差分**
- モック: 見出しがやや左寄り、行間はゆったり、文字色は柔らかい黒。
- 実装: 見出し位置・行幅・本文行間が少しズレている。

**再現する内容**
- h1の左端、上端、行間、文字サイズをモックに合わせる。
- ボタン群との縦距離を固定する。

**実装指示**
- `.hero-content` の `margin-left`, `padding-top`, `width` をPC基準で調整。
- `h1` の `font-size`, `line-height`, `letter-spacing` を比較しながら調整。
- 本文 `.lead` は横幅と行間をモックに合わせる。

### 3. CTAボタン
**差分**
- モック: 緑ボタン・白ボタンが横並びで、横幅・丸み・影が自然。
- 実装: ボタンサイズやアイコン位置が近いが、余白と影が少し違う。

**再現する内容**
- ボタン高さ、横幅、角丸、影、文字の太さを一致させる。

**実装指示**
- `.btn` height / padding / box-shadow を調整。
- `.primary` と `.secondary` の幅をモックに合わせる。

### 4. バッジ3つ・キャンペーン帯
**差分**
- モック: バッジの位置がCTA直下にきれいに並び、キャンペーン帯も自然な幅。
- 実装: 近いが、縦位置と横幅が少し違う。

**再現する内容**
- バッジの幅・高さ・gap・キャンペーン帯の幅を合わせる。

### 5. 右上フローティングカード
**差分**
- モック: 右上カードの位置・サイズ・角丸・影が写真と自然に馴染んでいる。
- 実装: カード位置・サイズは近いが、商品写真/装飾の密度が違う。

**再現する内容**
- カードのtop/right/width/heightを合わせる。
- カード内の文字サイズと余白を調整。

### 6. 下部安心バー
**差分**
- モック: ヒーロー下端に接続する白い安心バーの高さ・角丸・項目間隔が整っている。
- 実装: 高さ・位置・項目のテキスト密度が少し違う。

**再現する内容**
- `.trust-bar` のwidth / min-height / border-radius / item gapを合わせる。

### 7. SP版
**差分**
- PCモック基準からレスポンシブにしているため、SPは意図的に構成変更可。

**再現する内容**
- SPでは文字可読性優先。背景写真は薄め、CTAは縦積み、横スクロールなし。
- PC完全一致より、SP単体で自然なLP品質を優先する。

## 禁止事項
- プレビュー本体にモック画像を重ねて95%に見せること。
- 文字入り背景を使うこと。
- 同じ背景ファイル名を使い回すこと。
- 比較ページが見られない状態で完了報告すること。

## 完了条件
- 実URLプレビューで文字被りなし。
- 比較ページが200で開ける。
- 比較ページに「モック / 実コード / 50%重ね / 差分 / 改善指示」が載っている。
- PCスコアと目視差分を正直に報告する。

## 最新の改善試行結果
- フォント、もや、絵文字アイコンを個別に調整。
- アイコンは絵文字からSVGラインアイコンへ置き換え。見た目品質は改善。
- 数値スコアは約87.76%で横ばい。悪化はしていないが95%には未達。
- 次に稼ぐべき最大要因は、右側ヒーロー写真の構図（犬・商品・ボウル・植物・カードとの位置関係）。
- 95%到達には、Image2.0で「モック右側構図を維持した文字なしヒーロー写真素材」をさらに精密に作る必要がある。
